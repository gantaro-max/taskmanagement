# ER図

## エンティティ一覧

| テーブル名 | 説明 |
|----------|------|
| `boards` | ボード（タスク管理の大枠） |
| `lists` | リスト（ボード内のカラム。例: ToDo / Doing / Done） |
| `cards` | カード（個々のタスク） |

> 学習目的のシングルユーザー構成のため、`users` テーブルは含みません。
> 将来的に追加する場合は `boards` に `user_id` を追加します。

---

## ER図

```mermaid
erDiagram
    boards {
        INTEGER id          PK "自動採番"
        TEXT    title          "ボード名（必須）"
        TEXT    description    "説明（任意）"
        TEXT    created_at     "作成日時 (ISO8601)"
        TEXT    updated_at     "更新日時 (ISO8601)"
    }

    lists {
        INTEGER id         PK "自動採番"
        INTEGER board_id   FK "所属ボード"
        TEXT    title         "リスト名（必須）"
        INTEGER position      "表示順（0始まり）"
        TEXT    created_at
        TEXT    updated_at
    }

    cards {
        INTEGER id          PK "自動採番"
        INTEGER list_id     FK "所属リスト"
        TEXT    title          "カードタイトル（必須）"
        TEXT    description    "説明（任意）"
        INTEGER position       "リスト内の表示順（0始まり）"
        TEXT    created_at
        TEXT    updated_at
    }

    boards ||--o{ lists : "1つのボードに複数のリスト"
    lists  ||--o{ cards : "1つのリストに複数のカード"
```

---

## リレーション

| 親テーブル | 子テーブル | 種別 | 制約 |
|----------|----------|------|------|
| `boards` | `lists` | 1対多 | `ON DELETE CASCADE`（ボード削除でリストも削除） |
| `lists` | `cards` | 1対多 | `ON DELETE CASCADE`（リスト削除でカードも削除） |
