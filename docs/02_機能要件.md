# 機能要件

## 技術スタック

| レイヤー | 技術 | 理由 |
|--------|------|------|
| フロントエンド | HTML / CSS / Vanilla JS | フレームワーク不使用で基礎を学ぶ |
| バックエンド | Node.js + Express | JSで統一できる、学習コストが低い |
| データベース | SQLite（開発） → PostgreSQL（本番） | SQLiteはサーバー不要で始めやすい |
| 通信 | REST API（fetch API） | HTTPの仕組みを学べる |

### アーキテクチャ

```
ブラウザ (HTML/CSS/JS)
    ↕ HTTP (REST API / JSON)
Node.js + Express サーバー
    ↕ SQL
SQLite / PostgreSQL
```

## ボード機能要件

### B-1 ボード一覧表示
- 全ボードをAPIから取得して表示する（`GET /api/boards`）
- 各ボードに「タイトル」「カード総枚数」「削除ボタン」を表示する
- ボードが0件のときは空状態（U-2）を表示する

### B-2 ボード作成
- タイトルは必須入力。未入力で送信した場合は入力欄を赤枠にしてエラーを表示する
- 作成成功後にボード一覧を再描画する（`POST /api/boards`）
- `Enter` キーでも送信できる

### B-3 ボード削除
- 削除前に確認ダイアログ（U-4）を表示する
- 削除するとボード配下のリスト・カードもすべて削除される（DBのCASCADE制約で実現）
- 削除成功後にボード一覧を再描画する（`DELETE /api/boards/:id`）

## リスト機能要件

### L-1 リスト表示
- 選択したボードのリストとカードをまとめて取得する（`GET /api/boards/:id/lists`）
- リストは `position` 昇順で横並び表示する

### L-2 リスト作成
- タイトルは必須入力
- 作成後は既存リストの末尾に追加する（`POST /api/lists`）

### L-3 リスト名編集
- リストヘッダーのタイトル部分をクリックするとインライン編集に切り替わる
- `Enter` または確定ボタンで保存、`Escape` で変更を破棄する（`PATCH /api/lists/:id`）
- タイトル未入力での保存はできない

### L-4 リスト削除
- 確認なしで即削除する（カードも連鎖削除）（`DELETE /api/lists/:id`）

---

## カード機能要件

### C-2 カード作成
- タイトルは必須入力
- 作成後はフォームをリセットして連続入力できる状態にする（`POST /api/cards`）

### C-3 カード詳細表示・編集
- クリックでモーダルを開く
- タイトル（必須）と説明文（任意・複数行）を編集できる
- 保存ボタンまたは `Ctrl+Enter` で保存する（`PATCH /api/cards/:id`）
- モーダルに所属リスト名・作成日・更新日を表示する
- 背景クリックまたは閉じるボタンで未保存の変更を破棄して閉じる

### C-4 カード削除
- モーダル内の削除ボタン押下 → 確認ダイアログ（U-4）→ 削除（`DELETE /api/cards/:id`）
- リスト上の `[×]` ボタンは確認なしで即削除する

### C-5 カード移動（D&D）
- ドラッグ中はカードを半透明で表示し、元の位置に幽霊（グレー枠）を残す
- ドロップ可能なリストをハイライト表示する
- ドロップ後に `list_id` と `position` を更新する（`PATCH /api/cards/:id`）
- 同じリスト内での並び替えにも対応する

### C-6 カード移動（モバイル）
- カード詳細モーダル内にリスト選択のセレクトボックスを表示する
- 保存時に `list_id` を更新する

## APIエンドポイント一覧

| メソッド | パス | 機能ID | 説明 |
|---------|-----|-------|------|
| GET    | `/api/boards`            | B-1 | ボード一覧取得 |
| POST   | `/api/boards`            | B-2 | ボード作成 |
| DELETE | `/api/boards/:id`        | B-3 | ボード削除 |
| GET    | `/api/boards/:id/lists`  | L-1 | 特定ボードのリスト＋カード一覧取得 |
| POST   | `/api/lists`             | L-2 | リスト作成 |
| PATCH  | `/api/lists/:id`         | L-3 | リスト名更新 |
| DELETE | `/api/lists/:id`         | L-4 | リスト削除 |
| POST   | `/api/cards`             | C-2 | カード作成 |
| PATCH  | `/api/cards/:id`         | C-3,C-5,C-6 | カード更新（タイトル・説明・list_id・position） |
| DELETE | `/api/cards/:id`         | C-4 | カード削除 |

### レスポンス形式（共通）

```json
// 成功時
{ "data": { ... } }

// エラー時
{ "error": "エラーメッセージ" }
```

## バリデーションルール

| 対象 | フィールド | ルール |
|-----|----------|--------|
| ボード | title | 必須、1文字以上 |
| リスト | title | 必須、1文字以上 |
| カード | title | 必須、1文字以上 |
| カード | description | 任意、最大1000文字 |
